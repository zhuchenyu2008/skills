# 复习区与记忆卡片（不写进笔记正文）

目标：
- 你上传资料 → 生成学科笔记（在 `学习/`）
- 同时生成一份“可抽查”的记忆卡片（在 `复习/记忆卡片/`）
- 每晚定时或你手动触发“复习”，系统按记忆曲线出题并维护记录

## 目录位置

- 学科笔记：`学习/B 学科/...`
- 复习区：`复习/记忆卡片/`
- 复习调度与状态库（隐藏）：`复习/.openclaw/间隔复习/`

## 卡片文件命名

每篇学科笔记对应一个卡片文件（便于追溯）：
- `复习/记忆卡片/YYYY-MM-DD 学科 记忆卡片 章节或主题.md`

若同名冲突：尾部加 `（2）` `（3）`。

## 卡片文件格式（建议）

YAML（可选，但建议有来源）：
```yaml
学科: 数学
类型: 记忆卡片
日期: 2026-03-06
来源: [[学习/B 学科/数学/课堂笔记/2026-03-06 数学 课堂 ……|原笔记]]
```

正文：
```md
# 2026-03-06 数学 记忆卡片 ……

## 卡片
- 问题::答案
- 问题::答案
```

## 生成卡片的选题规则（像老师会考的）

优先覆盖：
- 定义（是什么）
- 性质/类别/归属（属于什么）
- 判断依据/适用条件（什么时候能用）
- 易错点（哪里容易错）
- 反例/边界（什么时候不行）
- 词义/词形/固定搭配/语法判断点/选项辨析等“最小记忆单元”

常见优先示例：
- 属于什么
- 性质是什么
- 判断依据是什么
- 对应哪一类
- 区别是什么
- 条件是什么
- 作用/后果是什么
- 词义 / 词形 / 固定搭配 / 语法判断点 / 选项辨析

优先避免：
- 材料方向
- 主题概括
- 趋势总结
- 过泛归纳
- 需要大段组织语言才能作答的题
- 明显偏聊天总结或方法口号式的题

说明：上面两组都只是经验性方向，不是硬白名单或绝对禁区；具体题目要按“是否短、准、能判对错、对提分有用、贴合该科该节内容”来判断。

方法类内容若确有必要，单独整理为“方法卡/套路卡”；默认不要混进普通 nightly 记忆卡。

每篇 6-12 条即可，宁少而精；一张卡只考 1 个点，若答案包含多个并列点就拆卡；题目要短、准、可抽查。

## 抽查与维护（概要）

- 定时：每天 20:30（北京时间）自动出题
- 手动：用户说“复习/开始复习/考我/抽查”立即出题
- 评分：由助理根据你的答案打分；你不同意可指出，我会按你反馈改分并重新更新记忆曲线
- **关键闭环**：只要新建/更新了 `复习/记忆卡片/*.md`，就必须立刻执行一次题库入库扫描：
  - `python3 <WORKSPACE>/skills/obsidian-spaced-recall/scripts/sr_cli.py --vault '<VAULT_PATH>' scan`
- 然后再做一次云同步；不要只生成卡片文件、不更新 `sr.sqlite`
