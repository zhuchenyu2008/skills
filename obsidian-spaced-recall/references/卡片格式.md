# 卡片格式（Obsidian 内联，便于扫描）

目标：格式足够“死”，脚本能稳定解析；同时对人类也友好。

## 1) Q/A（推荐）

一行一张卡：
- `问题::答案`

示例：
- 导数的几何意义是什么？::函数图像在该点切线的斜率
- 求单调区间的一般步骤是什么？::求导→解 f'(x)>0/<0→结合定义域→写区间
- `dark horse` 是什么意思？::黑马；出人意料的获胜者。
- `as to` 怎么翻译？::关于；至于。
- `what` 引导什么从句？::宾语从句。

要求：
- 先看答案，再决定问法；**只看题面，用户就要知道自己到底该答“翻译/词义/分类/语法点/条件/判断依据”中的哪一种**
- 除 Cloze 挖空外，不要把单个词、短语、名词、材料标签直接裸放在左边当题目；Q/A 卡必须写成明确可作答的问题
- 问句要尽量像考试真的会考的落点；常见优先问法包括“属于什么 / 性质是什么 / 判断依据是什么 / 对应哪一类 / 区别是什么 / 条件是什么 / 作用/后果是什么”，但这些只是示例，不是白名单
- 一张卡只考 1 个点；若答案包含多个并列判断点，必须拆卡
- 优先避免明显偏聊天总结或方法口号式的问法（如“说明什么 / 体现什么 / 更突出的点 / 为什么 / 怎么理解 / 优先排查什么”），但不要机械按关键词一刀切
- 对词汇/短语卡，像大小写、空格、连字符这类轻微写法差异（如 `cutting edge` / `cutting-edge`）默认按同一考点处理，不要重复做两张
- 答案尽量短（1 句/1 行/1 个公式），避免整段复制

## 2) Cloze（挖空，适合公式/定理关键字）

- `... {{c1::被挖空内容}} ...`

示例：
- 三角形内角和是 {{c1::180°}}
- 若函数在区间上可导且 f'(x)>0，则函数在该区间 {{c1::单调递增}}

说明：
- 只支持 `c1`（MVP）；需要多空再扩展。

## 3) 放置位置（当前约定）

- 不写进学科笔记正文。
- 统一写入：`复习/记忆卡片/` 下的卡片文件。
- 卡片文件内用：
  - `## 卡片`
  - 然后用无序列表列卡片

## 4) 生成卡片的选题规则（让它像“老师会考的”）

优先覆盖：
- 定义（是什么）
- 性质/类别/归属（属于什么）
- 判断依据/适用条件（什么时候能用）
- 易错点（哪里容易错）
- 典型反例/边界（什么时候不行）
- 词义/词形/固定搭配/语法判断点/选项辨析等“最小记忆单元”

常见优先示例：
- 属于什么
- 性质是什么
- 判断依据是什么
- 对应哪一类
- 区别是什么
- 条件是什么
- 作用/后果是什么
- 词义 / 词形 / 固定搭配 / 语法判断点 / 选项辨析

优先避免：
- 材料方向
- 主题概括
- 趋势总结
- 过泛归纳
- 需要大段组织语言才能作答的题
- 明显偏聊天总结或方法口号式的题

说明：上面两组都只是经验性方向，不是硬白名单或绝对禁区；具体题目要按“是否短、准、能判对错、对提分有用、贴合该科该节内容”来判断。

方法类内容若确有必要，单独整理为“方法卡/套路卡”；默认不要混进普通 nightly 记忆卡。

每篇笔记 6-12 张即可；宁可少而精准，也不要堆砌。一张卡只考 1 个点，若答案包含多个并列点就拆卡；题目要短、准、可抽查。
