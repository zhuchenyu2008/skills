# Obsidian Study Notes（高中学习笔记）

目标：把课堂转写/作业/错题/复盘材料，落库成 **可复习、可检索、可长期积累** 的 Obsidian Markdown。

硬风格：
- 文件夹名 / 文件名 / 标题：**全中文**（必要单位、专有名词、LaTeX 命令不算违规）
- 笔记必须是“成品笔记”：不要写“待补/TODO/以后再说”一类占位
- **整条流程未完成前不要中途回用户进度**：遇到上传、长转写、后台等待、同步这类步骤时，要自己持续盯到“归档/笔记/必要扫描/同步”全部完成后再一次性回复；只有在确实卡住且必须用户介入时，才中途打断用户

## 目录结构（默认）

统一在 Vault 根目录下使用主目录：`学习/`。

另外新增一个独立复习区（不把卡片写进笔记正文）：`复习/记忆卡片/`。

在 `学习/` 下创建（可微调，但保持稳定）：
- `A 收件箱/`（学科无法判定时临时放这里）
- `B 学科/<学科名>/{课堂笔记,作业,错题,资料,阶段复盘}/`
- `C 考试与试卷/{月考,期中,期末,联考}/`
- `D 索引与总表/`
- `E 附件/{图片,音频,资料}/`（可选；优先把资料放进学科内 `资料/`）

复习区与记忆卡片的目录与维护规则：见 `references/复习区与记忆卡片.md`。

### 落库路径硬规则（避免放错）
- 学科类内容：`学习/B 学科/<学科>/...`
- 资料（PDF/讲义/截图/转写文件/参考文档）：`学习/B 学科/<学科>/资料/`
- 课堂笔记：`学习/B 学科/<学科>/课堂笔记/`
- 错题：`学习/B 学科/<学科>/错题/`
- 学科无法判定：先放 `学习/A 收件箱/`，后续再归档

## 命名规则（文件名/标题）

优先结构（空格分隔）：
`日期 学科 类型 章节或主题 知识点 题目来源`

常用：
- 课堂：`YYYY-MM-DD 学科 课堂 章节或主题`
- 作业：`YYYY-MM-DD 学科 作业 作业名称或来源`
- 错题（一题一页）：`YYYY-MM-DD 学科 错题 知识点 题目来源`
- 复盘：`YYYY-MM-DD 学科 复盘 主题` 或 `YYYY-第N周 学科 周复盘`

补全与冲突处理：
- 缺字段：尽量从转写/题干/图片中抽取；仍缺则用保守占位：`本节课主题待定` / `来源待补`
- 重名：末尾加 `（2）` `（3）`

## 每篇笔记必须包含的 YAML 元数据

```yaml
学科: 数学
类型: 课堂  # 课堂|作业|错题|复盘|索引
日期: 2026-03-05
章节: 选择性必修二
主题: 导数的几何意义
来源: 转写|音频|图片|讲义
状态: 初稿  # 初稿|已整理|已复习
错因标签: []  # 仅用于 错题
```

错因标签规范：见 `references/错因标签与用法.md`。

## 工作流（决策树）

### 0) 先识别是否是音频/录音文件（新增硬规则）

如果用户发来的是课堂录音、语音消息、音频文件，或你明确知道输入是 `mp3` / `m4a` / `wav` / `aac` / `opus` / `ogg` / `flac` 之类音频：
- 不要要求用户先手动转文字。
- 必须先 read 并遵循：`skills/sensevoice-local/SKILL.md`
- 然后用：`bash /root/.openclaw/workspace/skills/sensevoice-local/scripts/transcribe_file.sh <音频文件路径>` 先做转写。
- 该脚本默认走本地 CPU 限资源转写；不要擅自去掉默认资源限制。
- 转写完成后，把“原音频 + 转写文本”一起视为课堂输入，再继续下面的正常流程。
- 原音频文件应归档到对应学科 `资料/`；课堂笔记的 `来源` 写为 `音频|转写`。
- 如果转写失败，先简短报告准确错误；不要假装已经听写成功。

### 0.5) 用户要上传大音频文件（新增硬规则）

如果用户明确说出类似：`我要上传大文件` / `上传大音频` / `开上传页面` / `给我上传链接`：
- 不要让用户继续往 Telegram 里硬发大音频。
- 必须先 read 并遵循：`skills/sensevoice-local/SKILL.md`
- 然后按 `references/upload-workflow.md` 启动一次性上传页；默认应生成 `服务器 IP + 端口 + 随机路径` 的一次性上传链接，并先把该链接发给用户。
- 上传成功后，读取上传结果里的本地文件路径，再把该文件交给 `scripts/transcribe_file.sh` 转写。
- 转写完成后，把“上传的原音频 + 转写文本”一起当作课堂输入，继续本 skill 的正常归档/课堂笔记流程。
- 上传入口必须是一次性的：成功上传 1 个文件后自动关闭；若超时，要明确告诉用户超时了。

### 1) 识别输入类型（硬规则）

A. **资料类输入**（讲义/课件/PDF/图片资料/参考文档）
- 动作：只归档 → 放入对应学科 `资料/`
- 默认不自动生成课堂/错题（除非用户明确要求）

B. **课堂转写/课堂笔记类输入**（转写文本/文件/手写笔记整理/音频转写结果）
- 动作：
  1) 先把原始转写/原始材料归档到 `资料/`（保留原件）
  2) 再生成/更新一篇 `课堂` 笔记到 `课堂笔记/`（用模板）
- 写作要求：同主题资料尽量合并进同一篇课堂笔记（避免碎片化）

C. **错题类输入**（题目照片/题干文本/错解/解析）
- 动作：一题一页 → 生成 `错题` 笔记到 `错题/`（用模板）
- 优先：链接到对应课堂笔记（同知识点/同章节）

D. 其他（大纲/章节梳理/复习提纲）
- 动作：按用户需求生成 `复盘`

### 2) 选择模板
- 课堂 → `assets/课堂笔记模板.md`
- 错题 → `assets/错题模板（一题一页）.md`
- 复盘 → `assets/复盘模板.md`

### 3) 填写顺序
先把模板的固定小节填完整；再按内容补“扩展区”。

### 4) 联动与索引维护（优先做）
详见：`references/联动改稿与索引维护.md`。

## 质量标准（Quality Bar）

- 首屏放“最关键的可复习信息”（结论/要点/公式/错因一句话）
- 段落短、标题清晰、要点列表化
- 数学内容默认使用 LaTeX：行内 `$...$`，独立公式 `$$...$$`
- 模板内没有标序号的全部为可选内容，如果用不到或没有，直接不使用

### 可选：记忆卡片（供间隔复习/抽查）

默认规则：
- **数学**：生成课堂笔记、错题笔记后，默认**不生成**记忆卡片
- **其他学科**：生成课堂笔记、错题笔记后，默认**生成**记忆卡片
- 用户单次明确指令优先级更高；若用户说“这次不要卡片/这次要卡片”，则覆盖学科默认规则

生成记忆卡片时：
- **不要把卡片写进笔记正文**
- 卡片统一写入：`复习/记忆卡片/`（每篇笔记对应一份卡片文件，文件内包含来源链接）
- 卡片格式：`问题::答案` 或 `{{c1::挖空}}`
- 问法尽量贴近老师课堂表达；答案尽量短（1 句/1 行/1 个公式）
- **默认采用“最小记忆单元”**：一张卡只考 1 个判断点；若答案包含两个及以上并列判断点，必须拆卡。优先词义、词形、固定搭配、语法判断点、条件、易错点、选项辨析。
- 生成前先自问：**考试会这样考吗？答案能不能用短句/术语直接判对错？** 不能就不要写进记忆卡片。
- **原则优先，不设死名单**：先看这张卡是否短、准、能判对错、对提分有用、贴合该科该节内容；满足这些，即使不在常见示例里也可以生成。
- **常见优先示例**：词义、词形、固定搭配、语法判断点、条件、易错点、选项辨析，以及“属于什么 / 性质是什么 / 判断依据是什么 / 对应哪一类 / 区别是什么 / 条件是什么 / 作用或后果是什么”等常见应试问法；但它们只是高频示例，不是白名单。
- **优先避免**：材料方向、主题概括、趋势总结、过泛归纳、需要大段组织语言才能作答的题，以及明显偏聊天总结或方法口号式的问法（如“说明什么 / 体现什么 / 更突出的点 / 为什么 / 怎么理解 / 优先排查什么”）；但不要把这些词机械当成绝对禁词，要按具体题目判断。
- 方法类内容若确有必要，单独整理为“方法卡/套路卡”；默认不要混进普通 nightly 记忆卡。
- 如果一条卡片不够短、准、可抽查，就拆小；拆不小就不要写进记忆卡片。
- 生成新卡前，尽量避免和现有卡片重复；对词汇/短语卡，像空格、连字符、大小写这种轻微写法差异（如 `cutting edge` / `cutting-edge`）默认视为同一考点，不要重复做两张。
- 该格式用于与 `obsidian-spaced-recall` skill 联动做 nightly 定时抽查
- **新增硬规则：只要本次写入新建/更新了 `复习/记忆卡片/*.md`，在本次回复结束前必须立刻执行一次入库扫描：**
  - `python3 /root/.openclaw/workspace/skills/obsidian-spaced-recall/scripts/sr_cli.py --vault '/root/obsidian/全部' scan`
- **入库扫描视为同一轮落库流程的一部分**：生成/更新卡片后，先 `scan`，再执行一次云同步；不要只写卡片不入库。
- `scan` 之后，`obsidian-spaced-recall` 会把每张卡的**上次评分 / 上次复习 / 下次复习**自动回写到该卡片文件底部的“复习状态（自动生成）”区块；不要手改这一段。
- 若 `scan` 失败，必须在回复里明确写出“记忆卡片已写入，但复习题库入库失败（附原因）”，不要假装已经接入记忆曲线。

#### 错题的记忆卡片（新增硬规则）

- **非数学错题**：生成/更新错题笔记后，默认**自动生成记忆卡片**，不要等用户再提醒一次。
- 错题卡片优先写成“**错误选项 → 正确认知**”的最小记忆单元，而不是整题复盘。
- 一张卡片只放 **1 个判断点**；优先记录你这次真正错掉的选项内容。
- 默认不要扩写步骤、原因链、整题通用套路；除非用户明确要求。
- 数量尽量少：通常 **1-4 张** 就够。
- 典型写法：
  - `东正教堂::俄罗斯、东欧等东正教影响区。`
  - `纱丽服饰::印度、孟加拉等南亚地区。`
  - `大清真寺::西亚、北非等伊斯兰文化影响区。`
  - `人口短期快速增加::主要是迁入。`
- 如果用户已经明确给出希望记住的点，优先按用户给的粒度写，不自行加料。

## Markdown 排版规范（让笔记更好复习）

目标：用 Markdown 的“视觉层级”来降低复习成本。

硬规则：**模板/笔记正文里不要出现“提示语/括号说明/可选/需要时再用/无则删除”等元文本**。如果某块不需要，直接删掉该小节或留空即可。

- **加粗**：只给“结论/易错点/要背的规则/答案关键一句”加粗；不要整段加粗。
- *斜体*：用于“提醒/备注/反例/容易混淆点”的短语。
- `行内代码`：用于固定字符串/命令/文件路径/按钮名（例如 `ob sync`、`学习/B 学科/数学/`）。
- 列表优先级：
  - 讲清步骤 → 用 **有序列表**（1. 2. 3.）
  - 罗列要点 → 用 **无序列表**（-）
  - 层级关系 → 用 **缩进子项**
- 换行：段落尽量短（1-3 行）；长段落拆成要点。
- 分界线：用于分割“题目/解法/复盘行动”等大块内容（`---`）。
- 表格：只在“对比/总结/清单化信息”时用，避免大宽表。
  - 典型：公式对比、题型-方法、错因-对策。
  - 例：

```markdown
| 项目 | 结论 | 备注 |
|---|---|---|
| 定义 | **...** | *易混点：...* |
| 适用条件 | ... | ... |
```

- 引用块：用于“资料来源/原话/老师强调/关键提示”（`>`）。

## 同步（Obsidian Sync + Headless Sync）

当你确认：服务器已完成 Headless Sync（`ob`）登录与 vault 绑定，并且本次确实写入了 vault（含归档资料/生成或更新笔记）——则 **落库后立即触发一次同步**。

执行方式（优先脚本）：
1) `scripts/sync_safe.sh`（自动处理残留锁，推荐）
2) 或 `ob sync --path "<vault路径>"`

常见问题与一次性配置：见 `references/服务器同步脚本（sync_safe）.md`。
